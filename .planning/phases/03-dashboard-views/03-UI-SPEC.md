# Phase 3 — UI Design Contract

> Visual and interaction contract for the dashboard views phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom CSS in Jinja2 templates, CSS-in-template via `{% block extra_styles %}`) |
| Preset | not applicable |
| Component library | none — hand-coded components matching existing base.html patterns |
| Icon library | inline SVG only — no external icon library |
| Font | Inter (400, 500, 600, 700) — already loaded from Google Fonts in base.html |

**Source:** base.html design system — all new templates extend `base.html` and use its CSS custom properties.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge internal padding, dot indicator gap |
| sm | 8px | Card internal gap between label rows, button icon-text gap, table cell vertical padding |
| md | 16px | Card content padding (sides), table cell horizontal padding, section gap between elements |
| lg | 24px | Page shell horizontal padding, card outer padding, section vertical gap |
| xl | 32px | Space between page header and grid, between variable summary block and broken leads table |
| 2xl | 48px | Space between breadcrumb + page title block and first content section |
| 3xl | 64px | Reserved — topbar height (existing, do not change) |

Exceptions:
- Topbar height is 64px (inherited from base.html — do not override)
- Touch targets (scan buttons, pagination) must be minimum 44px tall (accessibility requirement from base.html `.gear-btn` pattern)
- Traffic light dot: 10px diameter, no spacing token — explicit `width: 10px; height: 10px`

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|

| Subheading | 16px | 600 | 1.4 | Card workspace names, campaign names in rows, variable names in summary block |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f5f7` (`--bg`) | Page background, outside-card space |
| Secondary (30%) | `#ffffff` (`--bg-el`) | Cards, topbar, table rows, breadcrumb bar |
| Accent (10%) | `#2756f7` (`--blue`) | Scan buttons (primary action), active pagination page number |
| Destructive | `#c33939` (`--red`) | Red traffic light dot for >10% broken, broken variable values in table, error state text |

Accent (`--blue`) reserved for:
- "Scan" / "Scan All" call-to-action buttons (primary button fill)
- Active pagination page indicator (filled background or underline)

**Health status colors (traffic light system — D-08/D-09):**

| State | Dot Color | Hex | Threshold |
|-------|-----------|-----|-----------|
| Green | `--green` | `#1a8a3e` | < 2% broken leads |
| Yellow | `--amber` | `#b87a00` | 2%–10% broken leads |
| Red | `--red` | `#c33939` | > 10% broken leads |
| Gray (no data) | `--tx3` | `#8a8a8e` | No QA scan run yet |

**Freshness indicator colors (D-18):**

| State | Color | Threshold |
|-------|-------|-----------|
| Fresh | `--green` (`#1a8a3e`) | Last checked < 5 minutes ago |
| Stale | `--amber` (`#b87a00`) | Last checked 5–15 minutes ago |
| Old | `--tx3` (`#8a8a8e`) | Last checked > 15 minutes ago |

**Broken variable value display:** Use `color: var(--red)` for values that are empty, null, or "NO". Use `color: var(--tx2)` for the label prefix (e.g. "cityName:").

## Component Inventory

### Traffic Light Dot

```
.health-dot {
  display: inline-block;
  width: 10px;
  height: 10px;
  border-radius: 50%;
  flex-shrink: 0;
}
.health-dot--green { background: var(--green); }
.health-dot--yellow { background: var(--amber); }
.health-dot--red { background: var(--red); }
.health-dot--gray { background: var(--tx3); }
```

- Always paired with a text label — never stand-alone (accessibility)
- Appears left of workspace name on overview cards, left of campaign name on workspace rows

### Workspace Card (Overview Page)

Layout: white card (`--bg-el`), `--sh` shadow, `8px` border-radius, `24px` padding.

Content layout (top to bottom):
1. Row: `[traffic-light-dot] [workspace name at 16px/600]` + right-aligned `[Scan button]`
2. Row: broken percentage + raw count (e.g. `3.2% broken (12 / 375 leads)`) — 13px/500, `--tx2`
3. Row: campaign count — 13px/400, `--tx2` (e.g. `4 campaigns`)
4. Row: freshness timestamp — 13px/400, freshness color (e.g. `Last checked 3 min ago`)

Full card is a link (`<a href="/ws/{name}">`) — except the Scan button which is a separate `<button>`.

### Workspace Page — Campaign Row

Layout: table-style rows inside a white card. Each row:

Columns:
1. `[dot] Campaign name` — 14px/600, `--tx1`; dot uses health threshold logic
2. Status badge — `Active` (green pill) or `Draft` (gray pill) — 12px/500
3. Broken leads — `3.2% (12/375)` — 14px/500; red if >0, green if 0
4. Variables affected — `cityName, niche` — 13px/400, `--tx2`; truncated at 2 names + `+N more`
5. Last checked — freshness color, 13px/400
6. Actions — `[Scan]` button

Row hover: `background: var(--bg-hov)` (already in base.html).
Row is a link to `/ws/{name}/campaign/{id}`.

### Campaign Detail Page

**Variable Summary Block** (top section):

White card, 24px padding. Heading: `Variable Issues` (16px/600).

Each variable row:
- Variable name in backtick-style label: `cityName` (13px/500, `--tx1`)
- Count: `12 leads broken` (13px/400, `--tx2`)
- Visual bar: thin gray background bar with red fill proportional to `count / total_leads` — 4px height, 8px border-radius
- Sorted: most broken first

**Broken Leads Table** (below variable summary):

White card. Heading: `Broken Leads` (16px/600) + lead count badge.

Columns:
1. `Email` — 14px/400, `--tx1`
2. `Broken Variables` — shows each broken variable and value: `cityName: [empty]` or `niche: NO` — 13px/400; variable name in `--tx2`, value in `--red`
3. `Lead Status` — text badge: `Active`, `Contacted`, `Bounced` — 12px/500

Pagination: 25 leads per page. Classic numbered pagination: `← 1 2 3 4 5 →`. Active page: filled `--blue` background, white text. Other pages: `--bg-el`, `--tx1` text.

### Breadcrumb Bar

Positioned at top of page content area, 16px vertical padding, white background (`--bg-el`), border-bottom `1px solid var(--bd)`.

Content: `All Workspaces > Enavra > Campaign A` — each segment at 13px/500. Inactive segments: `--blue`, underline on hover. Current page segment: `--tx1`, not a link.

Separator: ` > ` at `--tx3` color.

### Scan Button

Two variants:

**Primary (Scan All / topbar):**
```

font-weight: 600

border-radius: 8px
min-height: 44px

**Secondary (per-workspace, per-campaign):**
```

Font: 11px/600. Padding: 2px 8px. Border-radius: 12px.

## HTMX Interaction Contract

### Scan trigger (D-16/D-17)

All scan buttons use HTMX to post to scan endpoints and then refresh data:

| Button | `hx-post` | `hx-target` | `hx-swap` | After trigger |
|--------|-----------|-------------|-----------|---------------|
| Scan All (topbar) | `/api/scan/all` | `#workspace-grid` | `outerHTML` | Replace workspace card grid with fresh render |
| Scan Workspace | `/api/scan/ws/{name}` | `#campaign-table` | `outerHTML` | Replace campaign table with fresh render |
| Scan Campaign | `/api/scan/ws/{name}/campaign/{id}` | `#campaign-results` | `outerHTML` | Replace variable summary + broken leads with fresh render |

HTMX response: Server renders and returns the partial HTML fragment. 200 status = success, replace target. 500 = show toast error via HTMX after-request event.

Spinner during scan: Use `hx-indicator` attribute pointing to a sibling spinner element. Show spinner, disable button. Remove on response.

### Pagination

Pagination links use `hx-get` with `?page=N` query param. Target: `#broken-leads-table`. Swap: `outerHTML`. No full page reload.

## Responsive Behavior (UX-03)

Desktop-first. Breakpoints:

| Breakpoint | Layout |
|------------|--------|
| ≥ 1024px (desktop) | 3-column workspace card grid; full campaign table with all columns |
| 768px–1023px (tablet) | 2-column workspace card grid; campaign table hides "Variables Affected" column |
| < 768px (mobile) | 1-column workspace card grid; campaign table collapses to stacked card layout per campaign |

On mobile, the campaign row collapses to a card with essential info: name, dot, broken count, Scan button. "Variables Affected" and "Last Checked" move into an expandable detail row.

The broken leads table on mobile: hide "Lead Status" column. "Broken Variables" column wraps to show variables stacked.

Breadcrumb bar on mobile: truncate middle segments with ellipsis if total length > viewport. Always show first and last segment.
