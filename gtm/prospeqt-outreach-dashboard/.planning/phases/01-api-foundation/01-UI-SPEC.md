# Phase 1 — UI Design Contract: Admin Panel (Workspace Management)

> Visual and interaction contract for the admin panel UI components in Phase 1. Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

---

## Context

This phase adds workspace management UI to the existing dashboard. The admin panel is accessed via a gear icon in the main nav bar and is restricted by password. It allows GTM engineers to add, view, and remove client workspaces (name + API key), and shows an error badge when a workspace's API connection is down.

**Design constraint from user feedback:** Luxury/high-end aesthetic. Prospeqt brand visual language (prospeqt.co). Simple table overview + drill-down over dense card layouts. No generic AI frontend aesthetics.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/JS — stdlib constraint) |
| Icon library | Inline SVG only (no external icon libraries — stdlib/no-CDN constraint) |
| Font | Inter (already loaded via Google Fonts in dashboard.html) |
| Source | Existing templates detected: admin.html, login.html, dashboard.html |

**Note:** The codebase uses `--no-deps` constraint (stdlib-only Python, no npm packages, no bundlers). All UI is vanilla HTML/CSS/JS inside server-rendered templates. The design system is a custom CSS variable token set already defined in dashboard.html.

**Existing token set detected in dashboard.html (light theme — use this as the base):**

```css
/* Detected from html[data-theme="light"] in dashboard.html */

--bd: #e0e0e4
```

**Design direction:** Apply Prospeqt brand aesthetic (prospeqt.co) — light background, generous white space, subtle card borders, luxury feel. The admin panel should feel like a polished internal tool, not a generic admin dashboard. Use the light theme token set as the base. Do NOT use the current dark theme from login.html and admin.html — those are being replaced.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f5f7` (`--bg`) | Page background behind all surfaces |
| Secondary (30%) | `#ffffff` (`--bg-el`) | Admin panel card surface, login card, table rows, modal surface |
| Accent (10%) | `#2756f7` (`--blue`) | See reserved list below |
| Destructive | `#c33939` (`--red`) | Remove workspace button, error badge on workspace, error toast |

**Accent `#2756f7` reserved for:**
1. "Sign In" button fill (login form)
2. "Add Workspace" primary action button fill
3. Input border on focus state (`:focus` ring)
4. Password show/hide toggle link
5. Overridden-value indicator (left border on inputs with non-default values)

**Destructive `#c33939` reserved for:**
1. "Remove" action link/button text color
2. API error badge on workspace row (background: `rgba(195,57,57,0.08)`, border: `rgba(195,57,57,0.2)`)
3. Error toast background tint + border
4. Error message text (inline form validation)

**Semantic status colors (existing — preserve):**
- Green: `#1a8a3e` — API connection healthy indicator
- Amber: `#b87a00` — warning state (not used in admin panel Phase 1)

## Component Inventory

### 1. Gear Icon (Nav Bar Entry Point)

**Placement:** Right side of topbar, between theme toggle and refresh button (or at far right).

**Spec:**
- Size: 36x36px button, 8px border-radius
- Border: 1px solid `var(--bd)`
- Background: `var(--bg-el)`
- Icon: SVG gear (settings cog), 18x18px, color `var(--tx2)`
- Hover: background `var(--bg-hov)`, border `var(--bd-s)`
- Touch target: 44x44px minimum (CSS: use padding or min-height to hit 44px)
- Active/open state: icon color shifts to `var(--blue)`, border to `var(--bd-s)`
- No label text — icon only (engineers know what a gear means)

**Interaction:** Click opens admin gate (password prompt if not authenticated, admin panel if already authenticated in this session).

---

### 2. Password Prompt (Admin Gate)

**Behavior:** Shown as a centered card overlay (full-page, not a floating modal) if the user is not authenticated. URL: `/admin/login`. On success, redirect to `/admin`.

**Layout:** Centered card, vertically centered in viewport.

**Card spec:**
- Background: `var(--bg-el)` (`#ffffff`)
- Border: 1px solid `var(--bd)` (`#e0e0e4`)
- Border-radius: 12px
- Padding: 32px all sides
- Max-width: 360px
- Box-shadow: `var(--sh-md)` (`0 4px 12px rgba(0,0,0,0.08)` in light theme)
- Page background: `var(--bg)` (`#f5f5f7`)

**Heading:** "Admin Access" — 18px, weight 600, color `var(--tx1)`, margin-bottom 24px.

**Form:**
- Label: "Password", 12px, weight 600, color `var(--tx2)`, margin-bottom 6px
- Input: type=password, full width, background `var(--bg)`, border 1px solid `var(--bd)`, border-radius 8px, padding 10px 14px, font-size 14px, color `var(--tx1)`
- Input focus: border-color `var(--blue)`, no box-shadow glow
- Submit button: full width, margin-top 16px, padding 12px, border-radius 8px, background `var(--blue)`, color `#fff`, font-size 14px, weight 600, label "Sign In"
- Button hover: `opacity: 0.9` (no color change)

**Error state:** Inline below the button — 12px, color `var(--red)`, text: "Incorrect password. Try again."

### 3. Admin Panel Layout

**URL:** `/admin`

**Layout:**
- Shell: max-width 1100px, margin auto, padding 24px
- Topbar: flex, space-between, margin-bottom 28px
- Title: "Workspace Admin", 18px, weight 600, color `var(--tx1)`
- Sign-out link: right-aligned, 12px, color `var(--tx2)`, text "Sign out", href `/admin/logout`, hover color `var(--tx1)`

**Sections:** Each wrapped in a `.section` card:
- Background: `var(--bg-el)` (`#ffffff`)

- Border-radius: 12px
- Padding: 20px
- Margin-bottom: 20px
- Box-shadow: `var(--sh)` (subtle multi-layer shadow, light theme values)

**Section title style:** 13px, weight 600, color `var(--tx2)`, text-transform uppercase, letter-spacing 0.06em, margin-bottom 16px.

---

### 4. Add Workspace Form

**Placement:** Top section of the admin panel, titled "Add Workspace".

**Form fields:**
1. Workspace Name — text input, full width, placeholder "e.g. SwishFunding"
2. Platform — select dropdown (options: "Instantly", "EmailBison"), width auto
3. API Key — text input (not password type — engineers need to see it), full width, placeholder "Paste API key"

**Field layout:** Stacked vertically with 12px gap between fields.

**Labels:** 12px, weight 600, color `var(--tx2)`, margin-bottom 6px each.

**Input spec (shared):**
- Background: `var(--bg)` (`#f5f5f7`)
- Border: 1px solid `var(--bd)`
- Border-radius: 8px
- Padding: 8px 12px
- Font-size: 14px
- Color: `var(--tx1)`
- Focus: border-color `var(--blue)`, no glow

**Primary CTA button:** "Add Workspace"
- Padding: 10px 24px
- Border-radius: 8px
- Background: `var(--blue)` (`#2756f7`)
- Color: `#ffffff`
- Font-size: 14px, weight 600
- Disabled state: `opacity: 0.4`, `cursor: not-allowed` (disabled when any required field is empty)
- Hover (enabled): `opacity: 0.9`
- Loading state: button text changes to "Adding..." during async submission

**Validation:** Client-side — all three fields required before button enables. No inline red error text for empty fields (button disabled state communicates this). Server-side error shown as toast (see Toast spec below).

---

### 5. Workspace List

**Placement:** Second section of the admin panel, titled "Workspaces".

**Layout:** Table (matches existing admin.html pattern — engineers are comfortable with it).

**Column headers:**
- "Workspace" (left-aligned)
- "Platform" (left-aligned)
- "Status" (left-aligned)
- "" (empty — action column, right side)

**Header style:** 11px, weight 600, color `var(--tx3)`, text-transform uppercase, letter-spacing 0.05em, padding 8px 12px, border-bottom 1px solid `var(--bd)`.

**Row spec:**
- Cell padding: 8px 12px
- Row border-bottom: 1px solid `var(--bd)` (last row: no border)
- Alternating rows: `rgba(0,0,0,0.015)` on even rows
- Vertical-align: middle

**Workspace name cell:** 14px, weight 500, color `var(--tx1)`.

**Platform badge:** Inline after workspace name (or separate cell). 10px, color `var(--tx3)`, background `var(--bg)`, border 1px solid `var(--bd)`, border-radius 4px, padding 2px 6px.

**Status column — API healthy:**
- Dot: 8px circle, background `#1a8a3e` (green), margin-right 6px
- Text: "Connected", 13px, color `var(--tx2)`

**Status column — API error badge:**
- Pill badge: background `rgba(195,57,57,0.08)`, border 1px solid `rgba(195,57,57,0.2)`, border-radius 6px, padding 2px 10px
- Text: "API Error", 12px, color `var(--red)` (`#c33939`), weight 500
- No icon — text-only badge for clarity

**Remove action:** Text link in last column, 12px, color `var(--red)`, cursor pointer. Label: "Remove". No button chrome — text-only link.
- Hover: text-decoration underline
- Position: right-aligned in its cell

**Empty state (no workspaces configured):**
- Table body replaced with a single row spanning all columns
- Content: centered, 14px, color `var(--tx2)`, text "No workspaces added yet. Use the form above to add your first workspace."
- No icons or illustrations — text only

---

### 6. Remove Confirmation Dialog

**Trigger:** Clicking "Remove" on a workspace row.

**Pattern:** Browser-native `confirm()` dialog — NOT a custom modal. This is intentional given the stdlib/no-build constraint and the internal tool context. Engineers can handle a native dialog.

**Confirm message (exact copy):** `"Remove [Workspace Name]? This cannot be undone."`

**On confirm (OK):** Issue DELETE request, remove row from table, show success toast.

**On cancel:** Do nothing. Row remains.

**Note for executor:** Do not implement a custom modal dialog for this. `window.confirm()` is explicit and fast. A custom modal is out of scope for Phase 1.

---

### 7. Toast Notifications

**Position:** Fixed, bottom-right corner, 24px from bottom and right edges. z-index: 9999.

**Spec:**
- Padding: 8px 16px
- Border-radius: 8px
- Font-size: 13px
- Max-width: 320px
- Auto-dismiss: 3000ms

**Success toast:**
- Background: `rgba(52,199,89,0.15)`
- Color: `#1a8a3e`
- Border: 1px solid `rgba(52,199,89,0.25)`
- Text examples: "Workspace added.", "Workspace removed."

**Error toast:**
- Background: `rgba(195,57,57,0.12)`
- Color: `var(--red)` (`#c33939`)
- Border: 1px solid `rgba(195,57,57,0.2)`
- Text examples: "Failed to add workspace. Check the API key and try again.", "Remove failed. Refresh and try again."

## What This Phase Does NOT Include

The following are out of scope for this UI spec and should not be implemented:

| Out of Scope | Reason |
|-------------|--------|
| Custom modal dialog for remove confirmation | Browser `confirm()` is sufficient; no custom modal framework in stdlib-only context |
| Inline editing of workspace names | Not in Phase 1 requirements; add only |
| API key masking/show-toggle after save | Engineers do not re-view keys after save; out of scope |
| Per-workspace threshold overrides | Phase 5 ADMIN-02/05 scope |
| KPI target editing | Phase 5 ADMIN-02 scope |
| Global threshold editing | Phase 5 ADMIN-05 scope |
| Animated loading skeleton | Phase 7 FE-02 scope |
| Dark theme for admin panel | Current dark admin.html being replaced by light theme; dark mode toggle is dashboard.html Phase 7 |
