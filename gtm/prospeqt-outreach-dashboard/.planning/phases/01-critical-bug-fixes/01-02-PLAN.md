<objective>
Fix the backfill race condition (stale not_contacted overwrites) and bound all thread spawning to a ThreadPoolExecutor(max_workers=10).

Purpose: When backfill takes longer than the cache TTL, its stale results overwrite fresher data. Unbounded threading.Thread() spawning can exhaust system resources with many campaigns. Both are production stability bugs.
Output: Race-condition-free backfill with generation counters + bounded thread pool + regression tests.
</objective>
